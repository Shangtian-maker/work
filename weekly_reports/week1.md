## 4. 代码说明
- 下载代码：负责批量下载 PDF 招股书至本地。
- 解析代码：将 PDF 文件解析为 Markdown 文本。
- 定位代码：在 Markdown 中按关键词定位与融资事件相关的章节和段落。
- 抽取代码：解析定位章节，提取 VC/PE 融资事件信息，并生成 JSON。
- 校验代码：检查 JSON 是否符合 schema，生成 validation_log 和 error_cases。
- 如何运行：Python 脚本按顺序执行，先下载、解析、定位、抽取，最后校验。

## 7. 本周形成的规则
- 文件识别规则：
  按公司名、股票代码匹配 PDF 文件，去除多余空格、括号及“股份有限公司”等后缀。
- 章节定位规则：
  1.关键词或规则匹配：
    查找与融资历史、股权演变、历次增资、股权转让、股东情况相关的关键词：
        融资：融资历史、融资轮次、增资、募资；股权：股权结构、股东、持股比例、股权变动；转让：股权转让、股权交易。
  2.Markdown 层级处理：
    支持 #、## 等标题层级，确保提取到的段落上下文完整。
  3.至少保证提取一个相关章节：
    如果找不到任何匹配章节，记录日志并标注失败。
  4. 可选增强：
    支持正则或 NLP 模糊匹配，提高匹配灵活性；
    提取章节标题和正文分开，方便后续 PE/VC 信息提取。
  5.模糊匹配 / 正则匹配：
    除了精确关键词，也会匹配包含关键词的近义表达或变体；
    支持忽略大小写和空格差异；
    多层级标题上下文保留；
    提取章节时，会把父标题一起保存，确保上下文完整。
  6.段落优先匹配
    如果某个段落里出现了目标关键词，也会单独提取，方便后续分析。
- 关键词列表：
  融资、增资、股权转让、投资、投资方、机构投资者、私募、PE、VC、风险投资、财务投资者。
- JSON字段规则：
  1.必填字段：disclosed_round、total_investment_amount、investor_original_name、evidence_text等；
  2.未披露轮次填 "未披露"，推断轮次需填写 inferred_round 与 round_inference_basis；
  3.投资金额、每股价格、投前估值、投后估值分开；
  4.投资方原始名称保持原文，is_pevc 填 yes/no/uncertain；
  5.每条事件尽量包含 source_section 或 source_page。
