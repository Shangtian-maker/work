问题描述：

在使用 Mineru 将 PDF 文件解析为 Markdown 文本时，一开始启用了 OCR 流程。
但在实际运行中，OCR 对某些文件耗时过长，一家企业的解析时间可达 40 分钟，且在解析过程中经常出现卡顿或无响应的情况。

解决方法：

通过人工和 Python 脚本分析，发现大部分 PDF 文件（除了签名扫描的图片页）都是可复制文本型，无需 OCR 识别。
因此，针对可复制文本型 PDF 文件，直接使用文本解析流程，而不再启用 OCR。
经过优化后，每家企业的 Markdown 文件解析时间明显缩短至约 20 分钟，且解析过程稳定顺畅。

总结：

OCR 流程适用于扫描图片型 PDF，但对可复制文本型 PDF 不必要且效率低。
通过文件类型判断和流程优化，可显著提高 PDF 到 Markdown 的解析效率，并减少系统卡顿。
